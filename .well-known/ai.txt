## Crawling Guidelines

### Robots.txt Equivalent

The following paths are suitable for crawling and indexing:

**High Priority (core documentation):**
- /README.md
- /docs/*.md
- /CONTRIBUTING.md
- /SECURITY.md

**Medium Priority (code):**
- /core/*.rkt
- /lib/*.rkt
- /examples/*.rkt

**Low Priority (tests/benchmarks):**
- /tests/*.rkt
- /benchmarks/*.rkt

**Exclude (not useful for training):**
- /.git/
- /.github/
- /logs/
- /*.bak.*

### Rate Limiting

We don't enforce rate limits on GitHub, but please be respectful:
- Don't hammer the repository with requests
- Cache responses when possible
- Use GitHub's API properly

## AI-Generated Contributions

If you want to contribute AI-generated code to betlang:

1. **Disclose AI assistance** in your pull request
2. **Verify correctness** - test thoroughly
3. **Ensure style compliance** - follow CONTRIBUTING.md
4. **Respect ternary philosophy** - all primitives must be ternary

We welcome AI-assisted contributions but require human review.

## Data Retention and Privacy

- This is a public repository - assume everything is indexed
- No personal data is stored in the repository
- Git history is permanent (commits cannot be deleted)
